# デモ動画 台本・絵コンテ（60〜90秒）

**目的**：審査5軸のうち「AIエージェントが価値の中心」「実用性・体験価値」「ユーザビリティ」を、実際に動く画面で見せる。
**尺**：60〜90秒（審査員は多数を見る。最初の10秒で"何のプロダクトか"を必ず伝える）。
**投稿先**：YouTube 限定公開 → URLを ProtoPedia「動画」欄に貼付。
**素材**：本番アプリ `https://app.eventtune.link` の画面録画＋サンプルデータ。ナレーションは音声 or 字幕どちらでも可（字幕推奨・無音環境で見られるため）。

---

## 構成（1カット=1メッセージ）

### ① フック（0:00–0:10）
- **画面**：バラバラの Excel/CSV が並ぶデスクトップ → EventTune のチャット画面へ切り替わる。
- **字幕/ナレーション**：
  > 「展示会のあと、名刺の山と列がバラバラのExcel。整形に数日、フォローは全員同じテンプレ——EventTune は、その"イベント後の壁"をまるごと引き受けます。」

### ② つくる：投入（0:10–0:25）
- **画面**：名簿・アンケート・ブース記録・費用の複数ファイルを**チャットにドラッグ&ドロップ** → 取り込みプランが提案され、承認。
- **字幕**：
  > 「バラバラのファイルを、そのまま投げ込むだけ。AIエージェントが列名や形式のゆらぎを読み解き、"意味"で統合します。」
- ここで一瞬、統合後のデータ（Person が Event に参加、Product に関心）が紐づく様子を見せる。

### ③ まわす／実用：分析（0:25–0:45）
- **画面**：チャットで「先月のイベントと比べて、費用対効果は？」と入力 → **AIが生成したPythonコード → 実データ上で実行 → 表・数値の結果**が表示。
- **字幕**：
  > 「振り返りも費用対効果も、チャットで聞くだけ。"それっぽい数字"ではなく、実データ上でPythonを実行して、コードと結果の両方で答えます。」

### ④ とどける：100人に100通り（0:45–1:05）
- **画面**：「来場者を"刺さる軸"で分けてフォロー案を作って」→ AIがセグメント軸を提案 → **人が承認（HIL）** → 各人の個別フォロー文面が生成 → 各ブロックに **`reason_for_inclusion`（なぜこの人にこの内容か）**。
- **字幕**：
  > 「参加者を刺さる軸で分けて、一人ひとりに最適化。どの一文にも"なぜこの人にこの内容か"の根拠つき。承認してからCSVで配信ツールへ。」

### ⑤ クローズ（1:05–1:20）
- **画面**：`architecture.png`（構成図）を数秒 → ロゴ＋タグライン。
- **字幕/ナレーション**：
  > 「Google Cloud の Vertex AI・Gemini・ADK、Cloud Run 上で動く、イベントマーケティング・インテリジェンス。」
  > （タグライン）**「イベントの熱が冷めないうちに、振り返りも、100人に100通りのフォローも届く。出会いを整え、成果へチューニングする。」**
- **エンドカード**：`EventTune` / `https://app.eventtune.link` / GitHub URL。

## 収録のコツ
- **最初の1カットで用途を言い切る**（審査員は冒頭で判断する）。
- 実際に**AIが自律的に判断・実行している瞬間**（コード生成→実行、軸の提案）を必ず映す＝「AIエージェントである必然性」の証拠。
- 個人情報はサンプルデータのみ。実顧客名は映さない。
- 画面録画は 1080p / 16:9。テロップは大きめ・短く。
- BGMを入れる場合は著作権フリー音源。
