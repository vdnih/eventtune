# プロダクト名・プロダクト概要の提案

[CURRENT_ISSUES.md](CURRENT_ISSUES.md) で指摘した通り、現名称 `marketing-mail-generator` は
①最もコモディティ化したカテゴリ（AI メール生成）を自称し、②自プロジェクトの命名規約
（[PHILOSOPHY_AND_NAMING.md](../PHILOSOPHY_AND_NAMING.md) §6 —— `generator` をシステム名に
使わない）にも違反している。これまでの全検討（ポジショニング・能力境界・戦略スコープ）を
踏まえ、コア価値を見つめ直した上で、プロダクト名と概要の案を提示する。

---

## 1. コア価値の再定義 — 名前が背負うべきもの

全ドキュメントの検討の合流点として、本プロダクトのコア価値は次の一文に集約される:

> **イベントで生まれた出会いを、意味に変え、根拠ある個別のはたらきかけに変え、
> イベントを重ねるほど賢くなる資産に変える。**

分解すると、名前が想起させるべき要素は3つ:

1. **出会い・縁**（イベントの人と人の接点が原材料である）
2. **意味・根拠**（ただの自動化ではなく、判断が見える知性である）
3. **蓄積・再会**（単発で終わらず、次につながり、複利で強くなる）

逆に、名前が想起させては**いけない**もの: メール／文面生成（コモディティ）、
イベント運営ロジ（Cvent の土俵）、汎用 AI（差別化の放棄）。

## 2. 命名要件

| # | 要件 | 根拠 |
|---|---|---|
| R1 | 「メールツール」「生成ツール」と誤認されない | POSITIONING §7・命名規約 §6（`generator`/`template` 禁止） |
| R2 | 上記コア3要素のいずれかを想起させる | §1 |
| R3 | 日本の BtoB 現場で呼びやすい（カタカナで安定して書ける・2〜4音節） | 主戦場が日本市場 |
| R4 | 製品カテゴリを添えて成立する（「◯◯ — イベントマーケティング・インテリジェンス」の形） | 新カテゴリ（SoI）を名乗るため、固有名＋記述ラベルの2層構成にする |
| R5 | 商標・ドメイン・既存サービスとの衝突は**別途要確認**（本書は未調査） | 実務上の必須手順 |

カテゴリの記述ラベルは全案共通で **「イベントマーケティング・インテリジェンス」**

「インテリジェンス（判断・根拠）」を名乗ることが、System of Intelligence という
ポジション（[STRATEGIC_SCOPE.md](STRATEGIC_SCOPE.md) §1)の宣言になる。

## 3. 候補案

### 案A: **Encore（アンコール）** — 縁・再会系

- **由来**: アンコール＝「観客の方から、もう一度を求められる」。イベントの出会いを、
  こちらが追い回すのではなく相手に望まれる再会（商談）へ変える、という理想を一語で表す。
  綴りの中に **En（縁）× Core（不変のコア＝Static Core 思想）** が埋め込まれており、
  設計思想（[MARKETING_PHILOSOPHY](../MARKETING_PHILOSOPHY.md)）とも響き合う。
- **タグライン**: 「その出会いに、アンコールを。」
- **概要文**: Encore は、展示会・セミナーで生まれた出会いを次の商談に変える
  イベントマーケティング・インテリジェンスです。バラバラのリスト・アンケート・接客メモを
  AI が意味で統合し、一人ひとりに「なぜこの内容か」の根拠つきでフォローを組み立てます。
  イベントを重ねるほど、あなたのチームのデータは賢くなります。
- **強み**: 発音・記憶とも最良（R3）。ポジティブな情緒。EN-CORE の二重の意味。
- **弱み**: 一般語ゆえ商標・ドメインが混みやすい（R5 要調査）。音楽系サービスとの連想。

### 案B: **Afterglow（アフターグロー）** — 熱・余韻系

- **由来**: 日没後も空に残る残光。全資料で使ってきた「イベントの熱が冷める前に」という
  中心メッセージを、そのまま名前にする。プロダクトが所有する時間（イベントの直後）を
  名前が指し示す。
- **タグライン**: 「イベントの熱を、成果に変える。」
- **概要文**: Afterglow は、イベント直後の「熱」を逃さないための AI プラットフォームです。
  終わった瞬間から、データの統合・振り返り分析・全員分の個別フォローまでをチャットで完遂。
  熱が冷める前に、全員へ「あなた宛て」の一通が届きます。
- **強み**: プロダクトの存在理由（後工程・タイミング）が名前から即座に伝わる。情景が強い。
- **弱み**: 英単語として長め（R3 でやや不利）。海外でゲーム周辺機器等の既存ブランドあり（R5）。

### 案C: **Kotowari（理）** — 意味・根拠系

- **由来**: 「理（ことわり）」＝物事の筋道・根拠。本プロダクト最深の差別化である
  Auditable AI（すべての判断に理由がある）を、日本語の一語で背負う。
  「AI は信用できない」という市場の空気への正面からの回答。
- **タグライン**: 「すべての施策に、理（ことわり）を。」
- **概要文**: Kotowari は、AI の判断がすべて説明できるマーケティング・インテリジェンスです。
  誰をどう分けたのか、なぜこの文面なのか、この数字はどう計算されたのか——
  イベントデータの統合から個別フォローまで、根拠の見える AI がチームの意思決定を支えます。
- **強み**: 差別化の核（根拠・監査性）と完全に一致。エンタープライズ・稟議文化への訴求力。
  和名として唯一性が高く商標リスクが相対的に低い（R5 は要確認）。
- **弱み**: イベント文脈（出会い・後工程）が名前から伝わらない。海外展開時に発音の壁。

### 案D: **Tsumugi（紡）** — 蓄積・紡ぎ系

- **由来**: 「紡ぐ」＝バラバラの繊維（散在データ）から糸（意味）を作り、織り上げていく。
  1回のイベントで糸を紡ぎ、回を重ねて反物（資産）になる——オントロジー蓄積の複利
  （防御性の源泉、POSITIONING §4）を身体的なイメージで表す。
- **タグライン**: 「出会いを紡いで、資産にする。」
- **概要文**: Tsumugi は、イベントごとに散らばるデータを紡ぎ、チームのマーケティング資産に
  織り上げる AI プラットフォームです。取り込むほど、人と関心のつながりが濃くなり、
  次のイベントのフォローと分析が賢くなります。
- **強み**: 「使うほど強くなる」という最も守りやすい価値と直結。柔らかく日本市場に馴染む。
- **弱み**: BtoB SaaS としてはやや情緒的で、切れ味（速さ・成果）の印象が弱い。
  同名の和風ネーミングは他業種に多い（R5 要確認）。

### 参考案E: 記述名に振り切る — **EventCortex / イベントマーケティング・インテリジェンス**

固有名を立てず「何であるか」で名乗る保守案（例: EventCortex＝イベントの「大脳皮質」）。
説明コストは最小だが、記憶に残らず商標防御もしにくい。**新カテゴリを自ら名乗って
market を教育する**という PMM 方針（R4）に照らすと、固有名（A〜D）＋記述ラベルの
2層構成が優る。比較のための参考として残す。

## 4. 推奨 — 案A: Encore

推奨は **Encore（アンコール）— イベントマーケティング・インテリジェンス** である。理由:

1. **ポジショニングステートメントとの一致**: POSITIONING §3 の一言は「イベントで会った
   全員に、根拠ある個別フォローを、翌日に。」——その先にある成果（相手から望まれる再会）を
   名前が指す。機能（統合・生成）ではなく**成果**を名乗る名前は、機能が模倣されても古びない。
2. **3要素のカバレッジ**: 出会い・縁（En）◎ / 蓄積・再会（encore の反復性）◎ /
   意味・根拠（Core＝Static Core）○ と、コア3要素を最も広く覆う。
3. **実用性**: 2音節・カタカナ安定・国内外で発音同一。「Encore で分析して」と
   動詞的に使える呼びやすさは日常利用の定着に効く。
4. **タグラインの接続**: 既存資料の「イベントの熱が冷める前に」（案Bの世界観）は
   Encore のメッセージングにそのまま吸収できる（逆は難しい）。

次点は **案C: Kotowari**。エンタープライズ市場を最初から主戦場にする戦略を取るなら、
「根拠の見える AI」を名前で背負う案Cが逆転する。**どの市場から立ち上げるかの意思決定と
セットで最終決定すべき**である。

## 5. 当面の実務対応

1. **商標・ドメイン・競合調査**（R5）を経てから正式決定する。本書の候補はすべて未調査。
2. **リポジトリ名の変更を推奨**: GitHub のリネームは旧 URL から自動リダイレクトされ
   コストが低い。決定した製品名（例: `encore`）または中立な記述名
   （例: `event-marketing-intelligence`）へ変更し、命名規約違反状態を解消する。
   ※ App Hosting / GitHub Actions（WIF）のリポジトリ連携設定は要追従確認（ADR-012）。
3. **名称決定前でも今すぐできる緩和策**: README 冒頭の一文を「メール生成ツールではなく、
   イベントマーケティング・インテリジェンスである」というポジション宣言に差し替える
   （現 README は既に「単なるメール生成ツールではない」と書いており、カテゴリラベルを
   与えるだけでよい）。
4. **ドキュメント上の表記**: 正式決定までは「Event Marketing AI Platform（仮称）」を維持し、
   決定後に一括置換する。

## 6. 商標・ドメイン調査（R5）— 「Event Cortex EnCore」案の予備調査結果

2026-07-02、Web 検索による簡易調査（正式な商標調査・弁理士確認の代替ではない）として、
参考案E「EventCortex」と案A「Encore」を組み合わせた候補「Event Cortex EnCore」を対象に
衝突有無を確認した。**結論: 前半・後半とも同一業界（イベント×AI）での重大な衝突があり、
この組み合わせでの採用は推奨しない。**

### 6.1 EventCortex（前半）— 致命的な衝突

`eventcortex.com` は既に稼働中の競合サービスに使用されている。運営元はスロベニアの
DA Solutions d.o.o.（ブランド名 HeapCortex）で、プロダクト名は **「EventCortex —
AI-Powered Event Management Assistant」**。本プロダクトが目指す「イベント×AI」と
**カテゴリが完全一致**しており、名称も同一。同社利用規約には "EventCortex" を自社の
商標・サービスマークとして明記し、無断使用を禁止する条項がある
（[EventCortex Terms of Service](https://eventcortex.com/legal/terms)）。
参考案Eはそもそも「比較のための保守案」だったが、この結果によりA〜Dいずれとの
組み合わせであっても **`EventCortex` を名乗ることは不可**と判断する。

### 6.2 Encore（後半）— 深刻な衝突

- **Encore Global**（旧 PSAV、ブランド表記 "Encore®"）: 「世界最大級のイベント制作会社」を
  自称し、AV・展示会・ハイブリッドイベント制作を85年・23か国で展開する業界大手。
  AI ツールをイベント業務に導入するニュースも出ており
  （[Encore Leverages AI Tools to Reshape Events for Gen Z](https://www.encoreglobal.com/about/newsroom/encore-leverages-ai-tools-to-reshape-events-for-gen-z/)）、
  **同一業界（イベント）における先行の登録商標（® 表示あり）**と評価できる。
  日本国内での商標登録有無は今回未確認（要追加調査）だが、グローバル展開時のリスクは高い。
- **一般語ゆえの過密**: このほか "Encore" を名乗る AI/SaaS プロダクトが多数存在する。
  Encore Cloud（AI コンテンツ管理、[encorecloud.ai](https://www.encorecloud.ai/product)）、
  Encore-AI（飲食店向け AI、[encore-ai.com](https://www.encore-ai.com/)）、
  Encore AI Labs（AI コンサル、[encoreai.com](https://encoreai.com/)）、
  Encore（バックエンド開発 PF、[encore.dev](https://encore.dev/)）、
  AWS Marketplace 上の "Encore" 製品、Adobe Encore（動画編集、旧製品）など。
  NAMING_PROPOSAL §3 案Aの弱み「一般語ゆえ商標・ドメインが混みやすい」は的中しており、
  `encore.ai` / `encore.com` 等の主要ドメインもほぼ確実に取得困難。

### 6.3 Cortex 単体の過密（補足）

"Cortex" は BtoB AI/SaaS 文脈で既に強いブランド連想が複数ある。
**Palo Alto Networks** が "CORTEX" を USPTO に商標登録済み（AI セキュリティプラットフォーム、
[Cortex - Palo Alto Networks Trademark Registration](https://uspto.report/TM/88979203)）。
**Snowflake Cortex AI** はデータ×AI 領域で広く認知されたエンタープライズ AI 機能ブランド
（[Snowflake Cortex AI](https://www.snowflake.com/en/product/features/cortex/)）。
いずれもエンタープライズ IT 購買層に強く刷り込まれており、BtoB SaaS の文脈で
"Cortex" を名乗ると本プロダクトと無関係な連想を招くリスクがある。

### 6.4 結論と推奨

「Event Cortex EnCore」は不採用とする。加えて、案Aの単独採用（Encore のみ）も
Encore Global との業界内衝突により**高リスク**と評価を格下げする。
[§4](#4-推奨--案a-encore) の推奨を見直し、**案C: Kotowari** または **案D: Tsumugi**
（和語・独自性が高く、商標衝突リスクが相対的に低いと本調査でも裏付けられた）を
軸に再検討することを推奨する。

**重要な留保**: 本調査は Web 検索による簡易確認であり、日本国内の J-PlatPat
（特許情報プラットフォーム）での正式検索、USPTO/EUIPO での正式区分別検索、
弁理士によるクリアランス調査は未実施。最終候補が1〜2案に絞られた段階で、
正式な商標調査（国内・主要展開国、第9類・第35類・第42類を中心に）を専門家に
依頼することを強く推奨する。

## 7. 決定 — EventWeave（2026-07-02）

§6 の調査を踏まえ、案A〜D・参考案Eのいずれでもなく、**「EventWeave」に確定した**。
「Weave（紡ぐ・織る）」は案D「Tsumugi」が狙っていた蓄積・複利のコア価値を、
発音・国際展開のしやすい英語一語で表現したものであり、§6.1 で致命的とされた
`EventCortex` のような同一カテゴリでの完全一致は確認されていない
（§6 で洗い出した周辺リスク——英国の休眠法人 `EventWeave Limited`、
`Weave Communications`（NYSE: WEAV）、`W&B Weave` 等——は残るため、
正式クリアランス完了までは商標的使用を伴う対外公開を保留する。§6 の留保を参照）。

## 8. 再決定 — EventTune（2026-07-02）

§7 で確定した「EventWeave」は、商標・ドメイン予備調査に加え、Firebase/GCP プロジェクト ID の
重複可能性を踏まえた最終検討の結果、**「EventTune」に変更する**。

「Tune（調整する・最適化する・調和させる）」は、イベントで生まれたバラバラな出会い（データ）
という不調和な状態から、AI が顧客との波長を合わせ、施策や分析の精度を継続的に「チューニング」
していくという本プロダクトのコア価値を体現する。§1 のコア3要素との対応は、出会い・縁
（Tune＝波長を合わせる）、意味・根拠（チューニングの精度＝判断根拠）、蓄積・再会
（重ねるほどチューニングされる）。

コピー・メッセージング一式は
[MESSAGING_EVENTTUNE.md](MESSAGING_EVENTTUNE.md) を参照。**注意**: 「EventTune」自体の商標・
ドメイン調査は本書 §6 の対象外（調査済みなのは旧名「EventWeave」のみ）であり、対外公開前に
別途実施が必要。ドメイン `eventtune.link` は AWS Route 53 で確保済み（2026-07-03）。

コピー・メッセージング一式は [MESSAGING_EVENTWEAVE.md](MESSAGING_EVENTWEAVE.md) を参照。
