## プロダクトビジョン

### ターゲットユーザー
BtoB SaaS・製造業・コンサルなど、**商談後のフォローアップメールを大量に書くマーケティング担当者**。特に以下の課題を持つ人:
- 顧客ごとに個別メールを書く時間がなく、テンプレートの使い回しになっている
- ChatGPT 等で生成すると「いかにもAI」な文章になり、ブランドトーンが崩れる
- 商談メモが散乱していて、まとめてメール化する作業に時間がかかっている

### 解決する課題
「商談メモ→メール化」のボトルネックを除去し、1人のマーケターが50件・100件分の個別カスタマイズメールを30分以内に生成できるようにする。

### 成功指標（ハッカソン評価軸）
- デモで「これは使える」と審査員に感じさせること
- アップロードから1件のメール生成まで3分以内
- 生成メールのクオリティが「汎用ChatGPTより明らかに高い」と感じられること

## スコープ管理

### ハッカソン MVP スコープ

#### P0（必須）
- [ ] Google Sign-In 認証（Firebase Auth）
- [ ] プロジェクト作成（ブランド名のみ設定）
- [ ] CSV ファイルのドラッグ＆ドロップアップロード（name, company, meetingNotes 列を想定）
- [ ] DataAnalystAgent: 完全性チェックと洞察抽出
- [ ] 情報不足コンタクト向け補完フォーム（静的フォーム、チャットUI不要）
- [ ] EmailWriterAgent: 1件につき1通のメールを生成
- [ ] メール一覧画面（カードUI）
- [ ] ワンクリッククリップボードコピー
- [ ] CSV エクスポート

#### P1（時間があれば）
- [ ] Firestore `onSnapshot` によるリアルタイム進捗表示（ポーリングから置き換え）
- [ ] 対話型補完チャットUI（ADK セッション使用）
- [ ] ブランドガイドライン設定UI（toneKeywords, avoidWords）
- [ ] 複数テンプレートの選択（Firestore に3件以上投入）
- [ ] メールカード上の品質スコア表示
- [ ] テーブルビュー切替

#### スコープ外（今回は実装しない）
- Excel (.xlsx) アップロード
- 外部MAツール連携・自動送信
- 課金・プランゲーティング
- チーム機能（マルチユーザープロジェクト）
- メール送信後の効果追跡（開封率・返信率）
- 分析ダッシュボード

## リスクレジスター

| # | リスク | 発生確率 | 影響度 | 対策 |
|---|--------|---------|--------|------|
| R1 | Vertex AI / ADK クォータ制限がデモ中に発動 | 中 | 高 | `gemini-2.0-flash` を優先使用（クォータ上限が高い）。ADK 登録が間に合わない場合は `google-generativeai` SDK へのフォールバックを用意 |
| R2 | ADK セッション管理の実装複雑性 | 高 | 中 | Day 1 は静的補完フォームで代替（P0）。ADK セッションは P1 として後回し |
| R3 | Cloud Run Cold Start によるデモ遅延 | 中 | 中 | デモ前日に最小インスタンス数を 0 → 1 に変更 |
| R4 | CSV 列名のばらつき（ユーザーが雑なファイルをアップロード） | 高 | 中 | DataAnalystAgent が Gemini で列名を柔軟にスキーマにマッピング。固定ヘッダーパースは使わない |
| R5 | Firebase App Hosting が対象リージョンで未対応 | 低 | 低 | `output: 'export'` に切り替え、Cloud Run でフロントエンドをホスト |
| R6 | 生成メールの品質がデモで審査員を納得させられない | 中 | 高 | 高品質テンプレートを5件以上事前投入。プロンプトを実際の商談メモで事前検証。デモ用サンプルデータを丁寧に準備 |

## 商用展開ロードマップ（ポストハッカソン）

### Phase 1: 商用 MVP（ハッカソン後 1〜6週）
- Stripe 連携によるサブスクリプション課金
- プランティア管理（Firebase カスタムクレーム + 月間生成件数上限）
- Excel (.xlsx) アップロード対応（`openpyxl`）
- `VertexAiSessionService` への移行（マルチインスタンス対応）
- OpenAPI スキーマから TypeScript 型自動生成（型安全な API 呼び出し）

### Phase 2: スケール（2〜4ヶ月）
- Cloud Tasks キューによるジョブのワーカー分離
- チーム機能（プロジェクト内の RBAC: owner / editor / viewer）
- Vertex AI Vector Search によるテンプレート RAG の高度化
- HubSpot / Salesforce からのコンタクトデータ直接インポート
- メールバリアント生成（1コンタクトにつき2案生成）

### Phase 3: エンタープライズ（5ヶ月〜）
- カスタム LLM ファインチューニング（Vertex AI supervised tuning）
- SOC 2 準拠・データレジデンシー（テナントごとの Firebase プロジェクト分離）
- ブランドチーム向けセルフサービステンプレート管理UI
- メール送信連携（SendGrid / SalesforceMarketing Cloud）+ 効果測定
